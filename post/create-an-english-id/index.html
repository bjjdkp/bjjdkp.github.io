创建一个英文id

2020-06-05
其他

Contents

  • huangzy1015
  • bjjdkp

因为喜欢安东尼，所以我一直使用的中文id是小铁瓜，不过仍旧一直想创建一个尽可能独一无二的英文id来使用。

为什么要用英文id，主要还是因为中文id的使用场景有限。比如邮箱地址不能用中文，很多站点注册id也不能用中文。

作为一个强迫症患者，我希望同一网站的注册账号和昵称能保持一致，同时所有网站使用同一id来保持统一性。所以要尽可能使英文id独一无二，避免某个网站注册时出现重复id，只能妥协。

huangzy1015

可能起英文id的思路有很多，我以前常用的思路基本就两种。

  • 其一是喜好球星的英文名，比如我用过 anthony, Canthony7, CAnthony7, Canthony777等。为什么有这么多变种，就是因为太大众化了，所以重复率很高。而且存在这样一个问题，人都是会变的，比如高中时喜欢艾弗森，不过上大学后喜欢安东尼，这就很尴尬了。
  • 其二是姓名拼音，同样会有很高的重复率，因为中国人口多，重名的也多。所以我一般会在后边加上生日来尽量保持独立性，最后就变成了 huangzy1015。这个id可以说陪我度过了整个大学阶段，那时主要用的站点以及常用邮箱都是用这个注册的，屡试不爽。

不过这种字母加数字的id不是很优雅，而且过于直观地暴露了自己的生日。

bjjdkp

至于再次更换id的契机，是在毕业后注册gmail邮箱，huangzy1015竟然已被使用，其实这种规则的id重复率应该极低，而且我也不记得之前曾注册过gmail邮箱。就算大学时注册过，长春的手机号也已经注销了，所以估计也找不回了。

所以就有了现在的id：bjjdkp，生成规则是将字母与数字做映射，这个映射规则可以随便制定，我只是简单的采取了0->a,1->b,2->c...这样的规则。然后可以根据喜好生成6位或8位（取决于月日映射为两个字母或四个字母）的id。

可能只有IT从业者才会从0开始计数，所以这种id产生重复的情景可能是，一个和你同年同月同日生的人，做了程序员，产生了这样生成id的想法，同时看到id后，能够忍受这样可读性极差的id。根据世界人口时钟，2020年06月05日，世界当日出生人数40W+（没有去考究权威性），所以出现id重复的概率还是很低的。当然了，如果能在生日上加上具体时间的话，将绝杀，可惜我不知道。

生成这个id之后，谷歌bjjdkp只有一页的搜索结果，于是就采用了这个id。

这样就避免了随时间流逝，一个人的心境和喜好的转变所产生的影响。 你喜欢的人可能会变，你的名字甚至也可以改，但出生日期是伴随一生不会变的印记。
